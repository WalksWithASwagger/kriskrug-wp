## Pagely cache gotcha (new this session)

`wp-admin → Pagely® → PressCACHE™ → "Purge page cache"` (`admin.php?page=press_cache`) clears the WordPress layer but does **NOT** invalidate the **ARES gateway edge** for the canonical URL — origin updates immediately, the public edge lags on its own TTL (refreshed overnight here). Confirm origin instantly with a cache-bust query (`?cb=$RANDOM`); for an instant public purge use the Pagely Atomic dashboard (atomic.pagely.com).
