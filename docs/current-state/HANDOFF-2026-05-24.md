## ⚠️ THE RULE THAT WAS CONFUSING AGENTS — read this

There are **two different ways to change the site. Use the right one:**

| Change type | How | Deploy? | Editable by KK? |
|---|---|---|---|
| **Global** — header, footer, homepage, single post, blog archive, design CSS/JS, the reveal fix | Edit theme files in the `aurora/v2` worktree → build zip → KK uploads in wp-admin (Replace) → purge Pagely cache | Yes (theme upload) | No |
| **Individual content pages** — Events, Services, the ~25 marketing/authority/multilingual pages | Write page **content via REST** (`POST /wp/v2/pages/{id}`), reusing the live theme's Aurora CSS classes | **No deploy** | Yes (wp-admin) |

**Decision (KK, 2026-05-24): content pages = DB content via REST, NOT theme templates.**
Do NOT build per-page theme templates anymore. Reuse live Aurora classes:
`aurora-speaking-hero`/`-copy`, `aurora-kicker`, `aurora-action-row`/`aurora-button`(`-primary`/`-secondary`),
`aurora-proof-section`, `aurora-section-heading`, `aurora-proof-grid`,
`aurora-proof-module`(`-text`), `aurora-proof-body`, `aurora-card-label`, `aurora-proof-tags`,
`aurora-proof-actions`, `aurora-final-cta`. **Always snapshot a page before editing**
(`?context=edit&_fields=…content.raw` → `backup/page-snapshots/`).

## Deploy mechanics & blockers

- **Theme deploy:** wp-admin → Appearance → Themes → Add New → Upload → Install Now →
  **"Replace current with uploaded"** (this final step has been missed repeatedly — watch it).
  Then Pagely® → **Purge All Caches + CDN**.
- **Content deploy:** REST POST to the page; then purge Pagely cache (REST edits don't auto-purge).
- **Pagely cache:** always purge after any change; verify with a `?cachebust=…` query.
- **SFTP/SSH: BLOCKED.** Key `~/.ssh/id_pagely` is registered (SSH user `client_claude`),

  port 22 appears IP-firewalled. Needs the exact host + likely an IP allowlist. Until then,
  no hands-off file deploys.

## Gotchas (have bitten us)

- **macOS clipboard is unreliable here** — gets overwritten between `pbcopy` and browser
  `cmd+v`. To fill a browser field, set value via `javascript_tool` + dispatch `input` event.
- **Don't rebuild content pages as theme templates** (the confusion this doc fixes).
- Local site `siteurl` can point at an unserved host → inner pages 301 to nowhere; set to
  `http://localhost:10003`.
