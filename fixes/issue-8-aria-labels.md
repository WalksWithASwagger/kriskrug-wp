# Issue #8 - ARIA labels for icon buttons

## Scope

Track A repo-side fix pack only. No live WordPress writes were made.

This issue affects the Catch Responsive header/footer controls and plugin-generated icon controls visible in the May 2026 public snapshots. Because this repo is not the live theme mirror, the safest deploy path is a sitewide Code Snippets enhancement rather than editing theme files on `main`.

## Snapshot evidence

Checked against:

- `docs/current-state/raw/homepage.html`
- `docs/current-state/raw/pages/*.html`
- `fixes/issue-5-color-contrast.css`
- `fixes/issue-9-button-hover-states.css`
- `docs/current-state/FIXES-LIVE-RECONCILIATION-2026-05-20.md`

Findings:

- Header social icon links are repeated across the raw snapshots as Genericons anchors with hidden text and `title`, but without explicit `aria-label`.
- Mobile menu, search toggle, and scroll-to-top controls are icon-first controls generated by the active theme.
- Popup Maker close buttons and most Jetpack carousel icon buttons already have labels, but the close label is generic and the image download control can render without a useful text label.
- Current theme CSS resets outlines in the critical CSS, so icon controls need an explicit visible focus treatment.

Representative raw HTML:

```html
<a href="#mobile-header-left-nav" id="header-left-menu" class="genericon genericon-menu">
    <span class="mobile-menu-text">Menu</span>
</a>
```

```html
<a class="genericon_parent genericon genericon-twitter" target="_blank" title="Twitter" href="http://www.twitter.com/feelmoreplants"><span class="screen-reader-text">Twitter</span> </a>
```

```html
<div id="search-toggle" class="genericon">
    <a class="screen-reader-text" href="#search-container">Search</a>
</div>
```

## Ready-to-apply snippet

Use `fixes/issue-8-aria-labels.php` as a Code Snippets plugin entry.

Deployment steps after backup/restore proof exists:

1. WordPress Admin -> Snippets -> Add New.
2. Name: `KK Issue #8 Icon ARIA Labels`.
3. Type: PHP snippet, run on site front-end.
4. Paste `fixes/issue-8-aria-labels.php`, removing the opening `<?php` if Code Snippets auto-wraps PHP.
5. Save inactive first, then activate during a low-risk window.

The snippet adds:

- `aria-label="Open primary menu"` and `aria-controls="mobile-header-left-nav"` to the mobile menu icon link.
- `aria-label="Open site search"` and `aria-controls="search-container"` to the nav search toggle link.
- `aria-label="Back to top"` to the scroll-up icon link.
- Concise social labels for Twitter, email, GitHub, YouTube, Instagram, and phone.
- A clearer popup close label and a fallback label for Jetpack image download.
- A 3px visible focus outline for the affected icon controls.

- Activate search with keyboard only. Focus must be visible before activation, and the search input must remain reachable.

- Open a Popup Maker modal and confirm the close button announces `Close popup`.

## WCAG mapping

- 2.1.1 Keyboard: affected controls remain native anchors/buttons and are keyboard reachable.
- 2.4.7 Focus Visible: explicit focus outline is added for icon controls.
- 2.5.3 Label in Name: labels preserve the visible/icon intent with concise action names.
- 4.1.2 Name, Role, Value: icon controls receive programmatic names without changing their native roles.
