## Search Console resubmission

Submit only:

```txt
https://kriskrug.co/sitemap.xml
```

Do not submit:

```txt
https://kriskrug.co/news-sitemap.xml
https://kriskrug.co/image-sitemap-index-1.xml
https://kriskrug.co/video-sitemap-1.xml
```

If those old sitemap URLs appear in Search Console, remove them if the UI allows it, otherwise ignore them and let Google age them out.

Recommended URL Inspection requests after sitemap resubmission:

```txt
https://kriskrug.co/
https://kriskrug.co/about/
https://kriskrug.co/blog/
https://kriskrug.co/work/
https://kriskrug.co/contact/
```

Then request indexing for the highest-value recent posts or updated topic hubs. Expect Search Console lag of 24-72 hours.

## Validation

Before or after any robots change:

```bash
curl -fsSL https://kriskrug.co/robots.txt
curl -I https://kriskrug.co/sitemap.xml
curl -I https://kriskrug.co/wp-sitemap.xml
curl -I https://kriskrug.co/news-sitemap.xml
curl -I https://kriskrug.co/image-sitemap-index-1.xml
curl -I https://kriskrug.co/video-sitemap-1.xml
```

Expected result:

```txt
/robots.txt lists only https://kriskrug.co/sitemap.xml
/sitemap.xml and /wp-sitemap.xml return 200
/news-sitemap.xml, /image-sitemap-index-1.xml, and /video-sitemap-1.xml return 404 and are not advertised
```

Validate syntax in Google Search Console's robots.txt tester after deployment.

## Rollback

Deactivate Code Snippets entry 7 only if the full SEO root-files behavior must be rolled back. That also disables the current `/llms.txt` and AI-crawler policy, so prefer editing the snippet rather than deactivating it.

No Search Console or Analytics settings are changed by this repo artifact.
